Question 1
(a) Define Operating System. Explain computer system structure.

An operating system is system software that acts as an interface between the user and computer hardware and manages system resources such as CPU, memory, and input/output devices.

A computer system consists of the following components:

Hardware: Includes CPU, main memory, and input/output devices which provide basic computing resources.

Operating System: Controls and coordinates the use of hardware among various application programs and users.

System Programs: Provide a convenient environment for program development and execution, such as compilers, assemblers, and shells.

Application Programs: Programs such as word processors, browsers, and media players that solve user problems.

The operating system acts as an intermediary between hardware and users to ensure efficient and safe use of system resources.

(b) Why do some systems store the operating system in firmware while others store it on disk?

Some systems store the operating system in firmware because firmware is non-volatile, reliable, and allows fast system startup. This is suitable for embedded systems where the operating system is small and rarely changed.

Other systems store the operating system on disk because modern operating systems are large and frequently updated. Storing the operating system on disk provides flexibility, easier updates, and supports complex functionalities required by general-purpose computers.

(c) What is an interrupt? How can we handle interrupt?

An interrupt is a signal generated by hardware or software that requests the immediate attention of the CPU. When an interrupt occurs, the CPU temporarily suspends the execution of the current program and transfers control to an interrupt service routine (ISR).

Interrupt handling process:

The interrupt signal is generated by a device or event.

The CPU saves the current execution state, including the program counter and registers.

Control is transferred to the appropriate interrupt service routine using the interrupt vector table.

The interrupt service routine executes and handles the interrupt.

After completion, the CPU restores the saved state and resumes the interrupted program.

(a) Explain the services provided by an operating system.

An operating system provides the following services:

Program Execution: Loads programs into memory, executes them, and terminates them properly.

Input/Output Operations: Provides a uniform interface for input and output operations and manages I/O devices.

File System Manipulation: Manages files and directories, including creation, deletion, reading, writing, and protection of files.

(d) Direct memory access is used for high-speed I/O devices to avoid increasing the CPU’s execution load.

(i) CPU–device coordination:
The CPU initializes the DMA controller by providing the starting memory address, direction of transfer, and the number of bytes to be transferred. The DMA controller then transfers data directly between the I/O device and main memory. Upon completion, the DMA controller generates an interrupt to notify the CPU.

(ii) Interference with user programs:
Yes, limited interference may occur. The DMA controller may temporarily take control of the system bus, causing brief delays in CPU memory access (bus contention). Cache coherency issues may also arise if the CPU cache contains outdated data. However, these effects are minimal compared to the performance benefits of DMA.

(b) What do you know about the critical section problem? How can we solve it?

The critical section is a part of a program where shared data or resources are accessed. The critical section problem is to design a mechanism that allows processes to access shared data without causing data inconsistency.

A correct solution must satisfy the following conditions:

Mutual Exclusion: Only one process can execute in the critical section at a time.

Progress: If no process is in the critical section, a waiting process must be allowed to enter.

Bounded Waiting: There must be a limit on how long a process waits to enter the critical section.

The critical section problem can be solved using mutex locks, semaphores, monitors, and hardware-based solutions such as test-and-set.

(a) Describe the necessary and sufficient conditions for deadlock to occur.

Deadlock is a situation in which a set of processes are permanently blocked because each process is holding a resource and waiting for another resource held by another process.

The necessary and sufficient conditions for deadlock are:

Mutual Exclusion: At least one resource must be held in a non-shareable mode.

Hold and Wait: A process holding at least one resource is waiting to acquire additional resources held by other processes.

No Preemption: Resources cannot be forcibly taken from a process and must be released voluntarily.

Circular Wait: A circular chain of processes exists in which each process is waiting for a resource held by the next process.

Deadlock occurs if and only if all these conditions hold simultaneously.

(c) Shortly explain the scheduling criteria for CPU scheduling algorithms.

The scheduling criteria for CPU scheduling algorithms are:

CPU Utilization: Percentage of time the CPU remains busy; should be maximized.

Throughput: Number of processes completed per unit time; higher is better.

Turnaround Time: Time from submission of a process to its completion; should be minimized.

Waiting Time: Total time a process spends waiting in the ready queue; should be minimized.

Response Time: Time from submission of a request to the first response; important for interactive systems.

Question 5
(a) Consider a computer system that runs 5,000 jobs per month with no deadlock prevention or avoidance. Discuss the arguments for and against installing a deadlock-avoidance algorithm.

(i) Arguments for installing deadlock-avoidance algorithm:

Installing a deadlock-avoidance algorithm prevents deadlocks and eliminates the need to abort and rerun jobs. This improves system reliability and avoids wastage of CPU time and operator effort. Since the system has 30 percent idle time, the additional 10 percent execution overhead can be accommodated without reducing throughput. Deadlock avoidance also improves system predictability and user satisfaction.

(ii) Arguments against installing deadlock-avoidance algorithm:

The deadlock-avoidance algorithm increases average execution time per job and increases turnaround time by about 20 percent. Deadlocks occur infrequently and affect only a small number of jobs, resulting in relatively low cost. Therefore, the added complexity and performance degradation may not be justified compared to the limited losses caused by occasional deadlocks.

(b) How can we recover from deadlock? Shortly describe.

Deadlock recovery techniques are applied after a deadlock has been detected. The main recovery methods are:

Process Termination:
One or more processes involved in the deadlock are terminated to break the deadlock. Processes may be aborted all at once or one at a time until the deadlock is resolved.

Resource Preemption:
Resources are forcibly taken from some processes and allocated to others. The system may roll back processes to a safe state to ensure correct execution.

(b) Define paging. Shortly describe address translation scheme with suitable example.
Paging is a memory management technique in which logical memory is divided into fixed-size blocks called pages and physical memory is divided into fixed-size blocks called frames. The operating system maintains a page table to map pages to frames, allowing non-contiguous memory allocation.
In paging, a logical address is divided into a page number and an offset. The page number is used to index the page table to obtain the corresponding frame number. The physical address is formed by combining the frame number with the offset.
Example:
If page size is 1 KB and the logical address is 2050, then page number = 2 and offset = 2. If page 2 is mapped to frame 5, the physical address is (5 × 1024) + 2 = 5122.

(c) Design a multithreaded program to count and print the total number of words in two files. Give two use-case scenarios.
A multithreaded program can be designed by creating two threads, where each thread reads one file and counts the number of words independently. Each thread updates a shared variable containing the total word count using proper synchronization. The main thread waits for both threads to complete execution and then prints the final total word count.
Use-case scenarios:
Parallel analysis of large log files to reduce processing time.
Document processing and indexing systems where word counts are needed from multiple files simultaneously.

(b) What is Belady’s Anomaly? Describe with example.

Belady’s Anomaly is a phenomenon in which increasing the number of page frames results in an increase in the number of page faults when using certain page replacement algorithms.

This anomaly occurs in the FIFO page replacement algorithm because it does not satisfy the stack property.

Example:
For the reference string
1 2 3 4 1 2 5 1 2 3 4 5,
FIFO produces 9 page faults with 3 frames and 10 page faults with 4 frames.
Thus, increasing the number of frames increases the number of page faults, which is known as Belady’s Anomaly.

(c) Explain demand paging with proper figure.

Demand paging is a virtual memory management technique in which pages are loaded into main memory only when they are required during execution. Pages that are not immediately needed remain on secondary storage.

When a process accesses a page that is not present in memory, a page fault occurs. The operating system fetches the required page from disk, loads it into a free frame in memory (or replaces an existing page), updates the page table, and resumes execution.

PREEMPTIVE VS NON PREEMPTIVE
In preemptive scheduling, the operating system can suspend a currently executing process and assign the CPU to another process based on priority or time slicing. This improves system responsiveness and is suitable for time-sharing and real-time systems but introduces higher overhead due to frequent context switching.

In non-preemptive scheduling, once a process is allocated the CPU, it continues execution until completion or blocking. This approach is simpler and has lower overhead but may result in poor response time and the convoy effect.

USER LEVEL VS KERNEL LEVEL THREAD
User-level threads are implemented and managed in user space without kernel involvement. Thread creation and context switching are fast, but a blocking system call blocks the entire process, and true parallelism is not possible.

Kernel-level threads are managed by the operating system kernel. Although thread operations are slower due to system calls, kernel-level threads allow true parallel execution on multiprocessor systems and ensure that blocking of one thread does not block the entire process.
